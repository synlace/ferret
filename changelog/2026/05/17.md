# [2026-05-17]

### Added
- feat(auth): mandatory password authentication for all Ferret instances
  - First-run setup wizard now requires a password (min 8 chars) before AI provider selection
  - Passwords stored as bcrypt hashes in SQLite `auth_credentials` table
  - Browser sessions issued as 24-hour `HttpOnly SameSite=Strict` cookies via `POST /api/auth/login`
  - `GET /api/auth/me` endpoint for session liveness checks
  - `POST /api/auth/logout` clears session from DB and cookie
  - Optional `FERRET_API_KEY` env var enables Bearer-token auth for programmatic/CI access
  - Global `require_auth` FastAPI dependency enforces auth on all routes; exempt paths: `/`, `/health`, `/api/setup`, `/api/auth/login`, `/api/ca-cert`
  - Setup reset (`DELETE /api/setup`) now also clears credentials and all sessions
  - New `/login` page in the UI with password field, show/hide toggle, and error feedback
  - `AuthProvider` context + `useAuth()` hook expose `logout()` to all UI components
  - Sidebar footer gains a **Sign out** button
  - `app-shell.tsx` redirects to `/login` on 401 from `/api/auth/me`
  - `conftest.py` updated: existing `client` fixture patches `require_auth` to no-op; new `authed_client` fixture seeds credentials and session for auth-specific tests
  - New `test_api_auth.py` with 14 tests covering login, logout, session cookie, Bearer token, expired sessions, and exempt paths
  - `justfile` `test api` recipe includes `test_api_auth.py`
  - `README.md` Authentication section documents browser login and Bearer token flows
- feat(auth): `PUT /api/auth/password` endpoint — change instance password (requires current password, min 8 chars, invalidates all sessions)
- feat(auth): Change Password section added to Settings page (order: CA Certificate → Change Password → AI Provider → Proxy)
- feat(auth): `lib/api-fetch.ts` utility wraps `fetch()` with `credentials: "include"` globally, fixing cross-port cookie delivery (UI port 3000 → API port 8000)
- feat(auth): `unauthed_client` fixture added to `conftest.py` — real `require_auth` enforcement with no pre-seeded session cookie, for rejection tests

### Changed
- fix(auth): session cookie changed from `SameSite=Strict` to `SameSite=Lax` to allow cross-port same-host requests (localhost:3000 → localhost:8000)
- fix(auth): root page (`/`) is now a client component that checks setup + auth before rendering, eliminating flash of content on fresh install
- fix(auth): `_AUTH_EXEMPT_PREFIXES` in `deps.py` covers all `/api/setup/*` paths so test-connection and reset work during first-run wizard without a session

### Fixed
- fix(tests): `test_api_auth.py` expanded from 14 to 37 tests — adds `TestChangePassword` (9 tests), broader `TestProtectedEndpointRejection` across 6 routers, and `TestExemptPaths` now uses `unauthed_client` (real enforcement, no cookie) instead of `authed_client` with `cookies={}` override
- fix(tests): `test_api_openrouter_keys.py` `app_client` fixture applies `dependency_overrides` for `require_auth`
- fix(tests): `test_api_security2.py` setup POST payloads include mandatory `password` field
- feat(auth): mandatory TOTP-based Multi-Factor Authentication (MFA)
  - SQLite schema extended with `totp_secret` and `mfa_enabled` columns (migration-safe `ALTER TABLE` guards)
  - `POST /api/auth/mfa/setup` — generates TOTP secret + QR code (base64 PNG) for authenticator app scanning
  - `POST /api/auth/mfa/verify-setup` — verifies first TOTP code and activates MFA
  - `POST /api/auth/mfa/challenge` — exchanges `ferret_pending` cookie + valid TOTP code for a full `ferret_session` cookie
  - `POST /api/auth/mfa/disable` — disables MFA (requires current password + valid TOTP code)
  - `GET /api/auth/mfa/status` — returns `{ mfa_enabled }` for the UI
  - Two-phase login: `POST /api/auth/login` now issues a short-lived `ferret_pending` cookie when MFA is enabled; `require_auth` rejects pending cookies on all protected routes
  - `pyotp` and `qrcode[pil]` added to `requirements.txt`
  - Login page extended with TOTP step 2: 6-digit OTP input, auto-submit on 6 digits, back-to-password link
  - Settings page extended with Two-Factor Authentication section: enable flow (QR modal with scan + verify), disable flow (password + code form)
  - `test_api_mfa.py` — 37 tests covering full MFA lifecycle, pending-cookie rejection, and regression of protected endpoints
  - `justfile` `test api` recipe includes `test_api_mfa.py`
- fix(ui): login page password and TOTP fields now re-focus after a failed attempt — `requestAnimationFrame` defers focus until React re-enables the input after `setLoading(false)`
